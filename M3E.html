M3E - Mind Map Model Engine M3E - Mind Map Model Engine

A Graph / Tree UI tool for framework-thinking in research and development. 科学研究・開発のためのフレームワーク思考を支援する Graph / Tree 構造 UI ツール。

M3E (Mind Map Model Engine) is a Graph / Tree UI tool aimed at supporting framework-thinking in scientific research and development. It is designed to become the next step beyond dialogue-form LLMs — visualizing and externalizing knowledge and thought, building a world model, and accelerating decision-making. M3E (Mind Map Model Engine) は、科学研究・開発のためのフレームワーク思考の支援を目的とした Graph / Tree 構造 UI のツールです。目標として対話形式の LLM の進化系となり、知識や思考を可視化・外在化し、世界モデルを構築し、意思決定を加速することを掲げます。

Its features include folder-like scope control, a conversion layer between linear structures (like this very paragraph) and tree structures, and three bands of thought — Flash / Rapid / Deep. Offline operation is the core; AI integration, team workflows, and cloud sync are currently under active development. 特徴として、フォルダーのようなスコープ制御機能、この文章のような線形な構造と木構造を相互につなぐ変換機能、Flash / Rapid / Deep の 3 つの思考の帯域を備えます。オフライン機能を核として、AI 連携・チーム運用・クラウド同期の機能を現在開発中です。

Vision Vision

Future Vision of M3E M3Eの究極の目標

V1

Scalable from daily notes to research 日常から研究までスケーラブルな知識ベース

Build a knowledge base where reliability accumulates not just locally but at long-term, large scale, without breaking down midway. 局所ではなく長期・大域でも信頼性が累積して、破綻しない知識運用を実現する。

V2

A workspace where humans and AI talk through structure 人間と AI が構造的に対話できる作業場

Share map / scope / node so that humans and AI can move work forward together within the same workspace. map / scope / node を共有しながら、人間と AI が同じ場で作業を進められるようにする。

V3

Crystallize thought into a world model and accelerate the projection cycle 思考・知識を世界モデルとして結晶化し、多様な成果物への射影サイクルを加速する

Treat the world model on the Deep side as an asset, and reach a state where Rapid outputs can be projected from it for specific purposes. Deep 側の世界モデルを資産にし、用途別の Rapid 出力を射影できる状態を目指す。

V4

Foundation that integrates Mapify × Miro × Obsidian × GitHub Mapify × Miro × Obsidian × GitHub を統合する基盤システム

Bind rapid mind-mapping, visual collaboration, structured knowledge, and version control into a single foundation operated as one. 迅速なマインドマップ化、ビジュアルコラボ、構造化知識、版管理を一つの運用に束ねる基盤を作る。

Treat tasks, strategy, status, handoff, and review together on a single map, so that humans and AI can advance development from the same view. task、strategy、status、handoff、review を map 上で一体に扱い、人間と AI が同じ視界のまま開発を進められる状態を目指す。

Core model 中核モデル

Flash / Rapid / Deep — one evolution axis Flash / Rapid / Deep — 単一進化軸

M3E handles the same underlying data across bands where granularity and structural properties evolve together — Flash / Rapid / Deep. These are not independent dimensions but a single axis of evolution. M3E は同じ基礎データを、単位（粒度）と構造の性質が連動して変化する帯域で扱う。帯域は Flash / Rapid / Deep の 3 段階。これは独立した複数軸ではなく、単一の進化軸。

Flash

Fragments and raw material (before structure) 断片・素材（構造化前）

Multimodal — text, images, audio, clips. Connected to daily life, sudden ideas, scattered notes. External analogue: inbox / clipboard / quick capture. マルチモーダル（テキスト・画像・音声・貼付）、日常との連結、突発的アイデア、散発的メモ。外部類似: inbox / clipboard / quick capture。

Rapid

One document — syntax tree 文書 1 つ — syntax tree

A self-contained logical structure as a single document, readable as natural-language exposition. External analogue: Freeplane mind-map / LaTeX outline / Markdown outline. syntax tree（親子、章立て、節、説明順序あり、線形化可能）。1 本の文書として閉じた論理構造、自然言語の説明として流し読める。外部類似: Freeplane mindmap / LaTeX 章立て / Markdown アウトライン。

Deep

A body of documents — semantic graph 文書群・知識体系 — semantic graph

A conceptual network spanning many documents, where the same node appears on multiple paths and the system is read as a whole. External analogue: nLab / Wikidata / mathlib4 dependency graph / Stacks Project / a researcher's mind. semantic graph（tree ではない、多種エッジ、説明順序とは独立）。複数文書を横断する概念網、同じノードが複数経路に現れる、体系として読む。外部類似: nLab / Wikidata / mathlib4 依存網 / Stacks Project / 研究者の頭の中。

Operations between bands run in three directions: promote (Flash → Rapid) takes raw material into a document structure; systematize (Rapid → Deep) bundles multiple documents into a conceptual network; project (Deep → Rapid) carves out a single document with an explanatory order for a specific purpose. Daily operation is the Rapid ⇄ Deep loop: systematize what you have written, then project a fresh document from the system for a new purpose. 帯域間の操作は 3 つの向き。昇格（Flash → Rapid）は素材を文書構造に取り込む。体系化（Rapid → Deep）は複数文書を束ね、概念網に編み上げる。射影（Deep → Rapid）は体系から特定目的の説明順序を持つ 1 文書を切り出す。日常運用は Rapid ⇄ Deep の往復であり、書いた文書を体系化して体系に取り込み、体系から新しい文書を射影する。
